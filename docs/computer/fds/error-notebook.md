# 编程错题本

这页用来统一记录我在不同知识点中的编程错误。  
它不是单纯收集“写错了什么”，而是按**错因**来分类，方便以后复盘时快速定位问题来源。

## 1. 分类方式

以后新增错题时，优先按下面几类归档：

### A. 语言语法与编译错误

适合记录：

- 保留字冲突
- 单双引号误用
- 少分号、少括号、类型写错
- 编译器直接报错的语法问题

### B. 函数、数组与接口调用错误

适合记录：

- 数组传参
- 指针与地址
- 函数声明与调用不一致
- 返回值理解错误

### C. 输入输出与流程控制错误

适合记录：

- `scanf` / `printf` 误用
- `%c`、空格、换行符读取问题
- `while` 条件写错
- 死循环、提前退出、漏判 `EOF`

### D. 数据结构与算法逻辑错误

适合记录：

- 根结点判断错误
- 合并条件写反
- 状态更新遗漏
- 边界条件错误

### E. 调试与复盘提醒

适合记录：

- 这类错误以后怎么快速排查
- 本题最值得记住的检查清单
- 一眼识别 bug 的经验

## 3. 专题一：C 语言并查集（Union-Find）易错点

这部分记录的是我在写 C 语言并查集时出现过的典型错误。

### A. 语言语法与编译错误

#### 1. 字符与字符串混淆

- 错误示范：`printf('yes');`
- 正确写法：`printf("yes\n");`
- 根本原因：
  - 单引号 `''` 只能表示单个字符，例如 `'y'`
  - 双引号 `""` 表示字符串，例如 `"yes"`
  - `printf` 需要的是格式字符串，而不是字符常量
- 下次提醒：
  - 看到 `printf`、`scanf` 这类函数时，先检查双引号是否写对

#### 2. 关键字冲突

- 错误示范：`void union(int S[], int root1, int root2)`
- 正确写法：`void UnionSet(int S[], int root1, int root2)`
- 根本原因：
  - `union` 是 C 语言保留关键字，表示共用体
  - 保留字不能拿来当函数名、变量名或结构名
- 下次提醒：
  - 函数名尽量用 `FindRoot`、`UnionSet`、`InitSet` 这类语义明确的名字

### B. 函数、数组与接口调用错误

#### 1. 数组作为函数参数时写法错误

- 错误示范：
  - 定义：`void Initial(S[], int n)`
  - 调用：`Initial(S[], N)` 或 `Find(S[], b)`
- 正确写法：
  - 定义：`void Initial(int S[], int n)`
  - 调用：`Initial(S, N)` 或 `Find(S, b)`
- 根本原因：
  - 定义函数参数时，必须写出元素类型
  - 调用函数时，数组名本身就代表地址，不需要再写 `[]`
- 下次提醒：
  - 定义时看“类型是否完整”，调用时看“数组名后面有没有多余的 `[]`”

#### 2. `scanf` 返回值理解错误

- 错误示范：`while (scanf(" %c", &a) != 'S')`
- 正确写法：`while (scanf(" %c", &a) != EOF && a != 'S')`
- 根本原因：
  - `scanf` 的返回值不是读到的字符
  - 它返回的是成功匹配到的输入项个数；如果成功读入一个字符，就返回 `1`
- 下次提醒：
  - 想判断读到了什么，比较变量本身
  - 想判断输入有没有失败，比较 `scanf` 的返回值

### C. 输入输出与流程控制错误

#### 1. `scanf("%c")` 读到空白字符

- 错误示范：`scanf("%c", &a)`
- 推荐写法：`scanf(" %c", &a)`
- 根本原因：
  - `%c` 会读取缓冲区中的任意字符
  - 上一次输入留下的空格或换行符也可能被直接读走
- 下次提醒：
  - 读取命令字符、操作类型时，优先写成 `" %c"`

#### 2. 输出时少写换行或格式串

- 错误示范：`printf("yes")`
- 更稳妥写法：`printf("yes\n")`
- 根本原因：
  - 不加换行不一定是错，但在调试或在线评测中，输出粘连会影响观察
- 下次提醒：
  - 调试输出优先带上 `\n`

### D. 数据结构与算法逻辑错误

#### 1. `Find` 的循环条件写错

- 错误示范：`for (root = x; root >= 0; root = S[root])`
- 正确写法：`for (root = x; S[root] >= 0; root = S[root]);`
- 根本原因：
  - 在双亲数组并查集中：
    - `S[i] < 0` 表示 `i` 是根
    - `S[i] >= 0` 表示 `S[i]` 是父结点下标
  - 所以继续向上找根的条件应当是“当前结点不是根”
  - 如果写成 `S[root] > 0`，当父结点下标恰好是 `0` 时会误判
- 下次提醒：
  - 0 下标并查集里，判断“是否还有父结点”时优先写 `>= 0`

#### 2. 按大小合并时比较对象错了

- 错误示范：`if (root1 > root2) { ... }`
- 正确写法：`if (S[root1] > S[root2]) { S[root2] += S[root1]; S[root1] = root2; }`
- 根本原因：
  - 应该比较的是集合规模，而不是根结点编号
  - 在按大小合并中，根结点存的是负数规模
  - `S[root1] > S[root2]` 说明 `root1` 的集合更小，应该挂到 `root2` 下面
- 下次提醒：
  - 看到“按大小合并”时，第一反应先看 `S[root]`，不要看 `root` 本身

#### 3. 合并前没有先找根结点

- 错误示范：`UnionSet(S, S[b], S[c]);`
- 正确写法：`UnionSet(S, Find(S, b), Find(S, c));`
- 根本原因：
  - 并查集的合并对象必须是两个集合的根
  - `S[b]` 和 `S[c]` 只表示当前父结点，不一定是最终根结点
- 下次提醒：
  - 只要看到 `Union`，就先检查参数是不是两个 root

### E. 调试与复盘提醒

#### 1. 先修第一条 `error`

- 经验：
  - C 语言编译报错常常是连锁反应
  - 第一条真正的 `error` 修好之后，后面很多报错会一起消失
- 下次提醒：
  - 不要被一大串报错吓住，从上往下先定位第一条最关键的语法错误

#### 2. 并查集提交前自检清单

- `printf` / `scanf` 的格式串是否用了双引号
- 是否误用了 C 关键字当函数名
- 数组传参时是否多写了 `[]`
- `%c` 前是否需要加空格
- `scanf` 返回值和读入变量是否分清了
- `Find` 是否真的找到了根
- `Union` 是否传入的是两个根
- 按大小合并时，比较的是不是 `S[root]`
